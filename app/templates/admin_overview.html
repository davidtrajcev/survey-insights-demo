Why this is intentionally small

For the case demo, the core value is privacy-safe reporting, org snapshots, and trends. This admin page proves the lifecycle without turning the demo into a full survey-builder clone.

Production integration point

In production, publishing would use an imported org snapshot from the authoritative people source and notify eligible employees through Slack, email, or an internal portal.

Presentation talking points

What this admin scope proves

  • P&C can create annual, half-year, or eNPS pulse cycles.
  • Each cycle is tied to one org snapshot, which protects historical reporting.
  • Publishing creates one participant eligibility record per employee in that snapshot.
  • Responses still stay anonymous because participant identity is not stored on response submissions.
  • Full drag-and-drop survey building is deliberately out of MVP scope.
